Fast.com Clone Project Structure

This document outlines the recommended project structure for the Fast.com clone project. Follow this structure for proper GitHub Pages deployment.

project/
├── index.html                # Main HTML file
├── css/                      # CSS stylesheets
│   └── style.css             # Main stylesheet
├── js/                       # JavaScript files
│   ├── speedtest.js          # Core speed testing functionality
│   └── language.js           # Language switching functionality
├── assets/                   # Static assets
│   └── images/               # Images and icons
├── locales/                  # Language files
│   ├── en.json               # English translations
│   └── it.json               # Italian translations
└── README.md                 # Documentation and deployment instructions

Important Notes for GitHub Pages Deployment

  • The index.html file must be at the root of the repository
  • All file paths in HTML, CSS, and JavaScript files should be relative
  • Ensure all files use the correct case sensitivity (GitHub Pages is case-sensitive)
  • The repository should not exceed 1GB in size
  • Follow the deployment instructions in the README.md file
